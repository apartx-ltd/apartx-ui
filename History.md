# История изменений — apartx-ui

## 2026-07-01

### Версия 0.1.38

### Изменено

* **`ChatMessageList`: горизонтальный отступ `px-2` по умолчанию.** Прокинутый потребителем `class`
  теперь мёржится поверх дефолта через `cn`/tailwind-merge (`cn('px-2', className)`), а не заменяет
  его — конфликтующая утилита потребителя выигрывает (`px-4`/`px-0` переопределяют). Раньше отступ
  приходилось задавать в каждом потребителе.

### Версия 0.1.37

### Изменено

* **Групповые пузыри: скругление углов по началу/концу серии.** `Message` получил соседа `next`
  и вычисляет `isGroupEnd` (helper `groupEnd`); у пузыря на стороне отправителя (справа у своих,
  слева у чужих) квадратится верхний угол, когда сообщение не первое в серии, и нижний — когда не
  последнее (`bubbleRadius`). Серия сообщений от одного автора читается как единый стек, а не набор
  одинаковых «капель». Флаг `isGroupEnd` также прокинут в слоты рендерера.

### Версия 0.1.36

### Изменено

* **Групповой чат: подпись автора у входящих сообщений.** Рендерер прокидывает в слоты флаги
  `isGroupStart`/`isGroupEnd`, а дефолтный header-слот показывает `authorName` только в начале
  визуальной группы у чужих сообщений (`authorName && isGroupStart && !me`) — раньше имя рисовалось
  на каждом сообщении при непустом `authorName`. DM-хост просто не заполняет `authorName`. Хосты со
  своим header-слотом (например админка) теперь получают `isGroupStart`, чтобы восстановить подпись
  отправителя в группах.

### Версия 0.1.35

### Добавлено

* **Кнопка «прокрутить вниз» в `ChatMessageList`.** FAB (chevron-down) в правом нижнем углу
  списка — появляется, когда прокрутили вверх дальше порога от низа, с бейджем количества
  непрочитанных входящих (`countUnread`); клик возвращает к низу и включает stick-to-bottom.
  `MessagesList` получил сигнал `onScrollAwayChange` (порог `scrollAwayThreshold`, по умолч. 100px).

### Версия 0.1.34

Новый модуль **`apartx-ui/chat`** — переиспользуемое ядро чата (Svelte 5, без Meteor/i18n) и его shell; выносится из cabinet/admin ради единого чат-клиента. Плюс серия фиксов паритета и QA по ходу адаптации в `apartx-admin` (0.1.27 → 0.1.34).

### Добавлено

* **Чат-ядро `apartx-ui/chat`.** Reducer оконной модели сообщений за швом `ChatTransport`
  (`fetchOlder`/`subscribeLive`/`sendMessage`/`markRead`); `ChatSession` (subscribe-then-fetch,
  оптимистичная отправка, `loadOlder`, `markRead`); slice `composer` (draft + reply-snapshot +
  debounced-persist через `DraftStore`); реестр рендереров по типу сообщения
  (`setMessageRendererRegistry`/`setDefaultSlots`/`registerChatDefaults`) — зеркало modal-registry;
  i18n-шов (`setChatI18n`) и чистые helpers (`deliveryTick`, группировка, `showDate`). Покрыто
  vitest, ноль зависимостей от Meteor/i18next.
* **Svelte-shell чата.** `ChatMessageList` (виртуализация поверх `virtua`, read-watermark с
  debounce, дивайдер непрочитанного, начальный скролл к непрочитанному), `Message` (группировка,
  дивайдеры даты/непрочитанного, hook контекстного меню `onContextMenu`/`menuOnClick`),
  `MessageInput` (autosize-textarea, Enter=отправка / Ctrl-Enter=перенос, `onTyping`, snippet'ы
  `leading`/`gate`, превью вложений), `MessageRenderer` (равномерный набор slot-пропсов +
  вычисляемый per-message `me`).
* **Шов `setSlotContext`** — хост инжектит непрозрачный объект, который раскладывается во все
  slot-компоненты, чтобы бизнес-рендереры получали `t`/`peer`/`chat` без связывания kit с Meteor.
* **`ChatSessionOptions.authorNameOf`** — резолвер имени автора для превью reply.
* **Центрированные сервисные сообщения (`type==='service'`)** — отдельная ветка в `Message`:
  системная заметка по центру (текст переводит хост), без пузыря.
* **`class`-passthrough у `ChatMessageList`** — прокидывается на скролл-контейнер (например `px-2`).

### Изменено

* **`markRead` принимает watermark-сообщение** (seq-or-createdAt), а не числовой `uptoSeq` —
  совместимо с сервером без `seq`.
* **Кнопка отправки — только иконка** (paper-plane); `sendLabel` теперь `aria-label`/`title`.

### Исправлено

* **Скроллбар не показывается на программных скроллах.** `VirtualList` больше не «раскрывает»
  нативный скроллбар при stick-to-bottom / scroll-to-index на монтировании — только на реальном
  пользовательском скролле (счётчик программных скроллов). Аналогично `Content`: программный
  scroll-restore (возврат назад к сохранённой позиции) флагает узел `__scrollRestoring`, а
  обработчик раскрытия его игнорирует.
* **Реактивность composer.** Черновик и reply читаются через `$derived` — программный
  `setDraft`/`setReply` (быстрый ответ, prefill reply из контекстного меню) теперь рендерятся:
  `value={composer.draft}` как plain-const компилировался в одноразовый атрибут и не обновлялся.

## 2026-06-30

### Версия 0.1.26

### Добавлено

* **`Lightbox` — переиспользуемый просмотрщик изображений (обёртка над `viewerjs`).**
  Декларативный Svelte 5 компонент: zoom / pan / rotate / клавиатурная навигация — из коробки
  от `viewerjs`, который полностью скрыт как деталь реализации. API: `images` (`{ src, alt? }[]`),
  `bind:open`, `bind:index`, `onClose`, плюс escape-hatch `options` (мёржится в `new Viewer`).
  - Живёт за отдельным сабпасом **`apartx-ui/lightbox`**; `viewerjs` добавлен в `dependencies`
    и **не** реэкспортится из корневого барреля — потребители без лайтбокса не тянут зависимость
    (как `swiper` за `apartx-ui/carousel`).
  - Ленивая (SSR-safe) загрузка `viewerjs` в `onMount`; `viewer.css` импортится внутри
    компонента — самодостаточно. z-index учитывает слой оверлеев (`getOverlayLayer()`):
    всегда поверх диалога, из которого открыт.
  - Рендерит только скрытую галерею; потребитель держит свои превью и по клику ставит
    `index` + `open = true`. Демо: роут `/lightbox`.
  **Использование:** `import { Lightbox } from 'apartx-ui/lightbox';`

### Версия 0.1.25

### Изменено

* **Убраны последние `<style>`-блоки из компонентов — keyframes переехали в общий CSS.**

    `kitIndeterminate` в `styles/animations.css`; разметка `animate-[kitIndeterminate_…]`.

## 2026-06-29

### Версия 0.1.24

### Изменено

* **`<style>`-блоки убраны из `Header`/`Footer`/`Toolbar` — safe-area теперь на Tailwind.**
  Per-component scoped-CSS давал лишний CSS-запрос в проде. Инсеты переписаны на TW
  arbitrary-классы (едут в общий app.css потребителя):
  - `Header`: `pt-[var(--safe-area-inset-top,0px)] [--kit-safe-top:0px]`;
  - `Footer`: `pb-[var(--safe-area-inset-bottom,0px)] [--kit-safe-top:0px]`;
  - `Toolbar`: `pt-[var(--kit-safe-top,var(--safe-area-inset-top,0px))]`; неиспользуемый
    класс-хук `kit-toolbar` убран.
  Каскад `--kit-safe-top` сохранён (custom-property ставится на сам элемент → наследуется в поддерево).

* **Keyframe-анимации overlay'ев вынесены из `<style>` в общий `styles/animations.css`.**
  `Dialog` (scrim + pop/sheet) и `Drawer` (scrim + slide) не выразить чистым TW (`@keyframes`,
  iOS-safe `rgb()`-scrim, `animation-fill-mode: backwards`), поэтому их CSS переехал из
  per-component `<style>` в новый plain-global файл — едет в едином app.css, без отдельного
  scoped-запроса на overlay. Подключён в баррель `styles/index.css`.
  **Миграция:** потребители, импортящие kit-стили поштучно (а не баррель `apartx-ui/styles`),
  должны добавить `@import 'apartx-ui/styles/animations.css';`.

## 2026-06-29

### Версия 0.1.23

### Изменено

* **Safe-area инсет переехал из `<Dialog>` в примитивы `Toolbar`/`Header`/`Footer`.**
  Откат хардкода v0.1.22 (padding на full-screen-панели `<Dialog>`): знание о safe-area
  теперь живёт в примитиве, который `Dialog` и так композирует, поэтому один механизм
  работает и в диалоге, и на обычной странице.
  - **`Toolbar`** владеет верхним инсетом: `padding-top: var(--kit-safe-top,
    var(--safe-area-inset-top, 0px))` (scoped `:global(.kit-toolbar)`, т.к. класс вешается
    через `cn()`). Покрывает доминирующий паттерн — голый `<Toolbar>` в полноэкранных модалках.
  - **`Header`** резервирует верхний инсет и зануляет `--kit-safe-top` для поддерева, чтобы
    вложенный `<Toolbar>` не дублировал отступ (двух-переменная модель).
  - **`Footer`** резервирует нижний инсет (home-indicator) и тоже зануляет `--kit-safe-top`.
  - **`Dialog`**: full-screen-панель больше не хардкодит padding; центрированная
    (non-fullScreen) зануляет `--safe-area-inset-top/bottom` для поддерева, чтобы
    `Header`/`Toolbar` внутри карточки не давали щель.
  На web/desktop переменные не заданы → 0, поведение не меняется. Затрагивает всех
  потребителей; admin/spaces безопасны (инсет-переменные не выставлены → 0).

## 2026-06-29

### Версия 0.1.22

### Исправлено

* **Полноэкранный `<Dialog>` не учитывал safe-area (статус-бар / home-indicator).**
  Контент `fullScreen`-диалога рендерится `absolute inset-0` внутри портала (вне `#app`

  как у `BottomNav`, который уже уважал нижний inset). На web/desktop переменные не заданы
  → 0, поведение не меняется. Затрагивает всех потребителей (admin/cabinet/spaces) —
  безопасно: где inset-переменные не выставлены, padding схлопывается в 0.

## 2026-06-29

### Версия 0.1.21

### Добавлено

* **History-back для модалок модал-реестра.** `<Dialog>`, отрендеренный реестром (внутри
  `<ModalLayer>` → присутствует overlay-layer контекст), теперь зеркалит своё `open` в

  — back и не-back закрытие не дёргают history дважды. Интеграция **гейтится** по
  `getOverlayLayer()`: standalone-`Dialog` (без реестра, напр. в spaces/admin) не затронуты.
  Единственный back-interceptor вешает `<ModalOutlet>` один раз на клиенте
  (`initOverlayStack`). Раньше модалки реестра вообще не реагировали на BACK (уходили в
  роутер страниц).

## 2026-06-28

### Версия 0.1.20

### Исправлено

* **Модал-реестр ремаунтил ВСЕ открытые модалки при любом изменении стека.**
  `<ModalOutlet>` ждал `registry[id].load()` инлайн в `{#each}` `{#await}`; открытие/
  закрытие вложенной модалки переподнимало лоадер (новый промис) и **ремаунтило** ветку
  `then` — молча сбрасывая локальный `$state` родительской модалки на полпути (напр.,
  `CartModal` терял выбранные даты в момент открытия пикера поверх него). Так как Svelte 5
  ремаунтит `then` даже при стабильном промисе на ре-рендере хоста, лечение двойное:
  (1) мемоизация промиса чанка по id в реестре (`loadModal`), (2) новый `ModalHost`
  (по одному на `m.key`) резолвит компонент **один раз** в `$state` и рендерит
  `{#if Comp}<Comp/>` (без инлайн-`{#await}`). Стабильная ссылка `Comp` ⇒ Svelte больше
  не ремаунтит модалку при изменениях соседних слоёв стека.

* **`DateRangePicker` не пробрасывал выбранный диапазон.** Сегменты заполнялись, но
  двунаправленный `$effect` между внутренним `DateRange` и связанной строкой `{start,end}`
  затирал себя — выбор в календаре не доходил до потребителя. Перевели на `onValueChange`
  bits-ui как единственный источник истины (value→range остался охранённым эффектом).

* **Календари date-пикеров открывались ЗА слоёным `Dialog`** (z-index) и **обрезались**.
  Контент `bits-ui` рендерился на `z-50` (тот же слой, что и `Dialog` реестра) — клики по
  дням/Next перехватывались. Чиним как у `Combobox`: читаем `getOverlayLayer()` и рендерим
  контент на `layer.z + 2`. Плюс `DateRangePicker` (в отличие от `DatePicker`) **не** был
  обёрнут в `<…Portal>` — попап обрезался `overflow` диалога; добавили `Portal`.

## 2026-06-27

### Версия 0.1.19

### Исправлено

* **`Combobox`: попап открывался ЗА слоёным `Dialog` модал-реестра.** Дропдаун
  портируется в `<body>` с `z-50`, но `Dialog` (обёрнутый в `<ModalLayer>`) рендерит

  глубокой модалкой); без слоя — прежний `z-50` (обратно совместимо, как `Dialog`/
  `BottomSheet`).
* **`Combobox`: значение не отображалось, если приходило ПОСЛЕ маунта** (асинхронный
  fetch опций, сидинг формы редактирования). Корень — `defaultValue` у `Combobox.Input`
  (bits-ui) не реактивный (применяется один раз при маунте), поэтому поздно пришедший
  `value`/`options` навсегда оставлял инпут на плейсхолдере. Перешли на реактивный бокс
  Root `inputValue`: пока попап открыт — показываем поисковый запрос, когда закрыт —
  label выбранной опции (реактивно; disabled-комбобокс тоже показывает значение).

### Версия 0.1.18

### Добавлено

* **`PageTransition`: проп `slideStyle` (`'auto' | 'ios' | 'shared-axis'`).** Осознанный
  выбор стиля анимации смены страниц вместо неявного дефолта. `'ios'` — горизонтальный
  push-слайд в стиле iOS (входящий слой выезжает поверх, уходящий слегка уезжает под
  него); `'shared-axis'` — Material shared-axis (слой целиком уезжает по оси с
  затуханием). `'auto'` (дефолт) резолвится по ОС через `detectMobileOS`: iOS → `'ios'`,
  остальное → `'shared-axis'`. Стиль прокидывается в DOM как `data-slide-style`, а
  keyframes заскоуплены по этому атрибуту — оба стиля работают рядом без протечек.
* **`detectMobileOS` (`apartx-ui/utils`)** — лёгкий детект мобильной ОС (iOS/Android/other)
  по UA, для резолва `slideStyle='auto'`. Покрыт первым Vitest-харнессом кита.
* **Экспорт типа `SlideStyle`** из `apartx-ui/navigation` для типобезопасного
  использования пропа потребителями.

### Исправлено

* **`shared-axis`: bleed/flash-free.** Затухание применяется ко всему слою (а не только
  к контенту) и на forward-in, и на back-exit — нет вспышки фона и «протечки» соседнего
  слоя на стыке транзишена.

## 2026-06-16

### Версия 0.1.17

### Изменено

* **`Image`: сессионный кэш загруженных картинок (`<script module>`).** Модульный
  `Set<string>` хранит `src`, которые хотя бы раз успешно загрузились. При повторном
  монтировании `<Image>` с уже известным `src` стартует сразу в статусе `loaded`
  (без skeleton, `opacity-100` с первого кадра) вместо повторного прохода
  `loading → loaded`. Раньше каждый ремоунт переигрывал skeleton + 300ms fade, хотя
  байты уже были в кэше браузера — это давало моргание (в списках, при навигации
  туда-обратно). Кэшируются только успешные загрузки, так что ошибки по-прежнему
  идут через повтор. SSR/гидрация не ломаются: на первом клиентском рендере кэш пуст
  (заполняется только после `onload`), выигрыш — на последующих ремоунтах.

## 2026-06-05

### Версия 0.1.2

`Toolbar` разносит элементы по умолчанию; длинные опции `Select` переносятся.

### Изменено

* **`Toolbar` — отступ и перенос между элементами из коробки.** Контейнер children получил `flex-wrap` + `gap-2`: раньше у него не было `gap`, и любой `gap-*`, переданный на сам `Toolbar` через `class`, на расстояние между детьми не влиял (разносил только `start | children | end`) — из-за чего несколько элементов слипались. Добавлены escape-hatch пропсы `startClass`/`contentClass`/`endClass` для точечного переопределения обёрток слотов (например `contentClass="gap-4"`).
* **`Select` — перенос длинных опций.** Пункты выпадающего списка (`min-w-0 break-words`, выравнивание по верху) переносятся на вторую строку вместо обрезки — длинные значения без пробелов (`lock_renew_subscription_agent_reward`) больше не вылезают за ширину панели.
